# PersonaChat Skill

> 支持根据消息感知和分配情感因子的类人聊天风格处理模块

## 设计初衷
当大语言模型的回答过于死板，或想让自动化工具的提示日志具有“人格色彩”时，便可通过此 Skill 做文本末端的风格化洗稿和渲染。

## 作为系统插件使用

在 OpenClaw 的执行空间中，可以通过内置终端命令非常直接地与之产生互动并捕获输出：

```bash
# 自动感知：
personachat "用户今天很伤心，说被老板辞退了" --persona INFJ
# 输出: "我懂你的...抱抱"

# 在垂直领域中：
personachat "我要挂号" --persona ENFJ --corpus medical
# 输出: "好的，请放心，马上帮您预约~💪"
```

## 功能演进

此项目目前已从原本纯随机抽卡的实验模型进化为具有**容错装载**以及能根据 `message` 的**字长/关键词进行情绪干预调度**的稳健版本。

## 配置与数据替换

您可以通过修改 `data/` 下的数据来训练出您自己的人格或团队专属提示：
- `personas.json` - MBTI人格及其固化用语定义
- `emojis.json` - 全局情绪标签与 Emoji 对应词库
- `corpus/` - 按业务/行业细分的专用话术库

## 触发场景策略参考

当用户或上游 Agent 意图表达：
- 希望冰冷的控制台、回复流变得更有"人味"时。
- 情感化服务（需要对客输出特定情感安抚回复）。
- 对于大量日常询问的口语化缓存支持。
